## 页面概述

凭证审核页（模块编码：`FinanceVoucherAudit`）是财务管理系统的凭证处理中心，供会计、出纳、财务主管对已录入的凭证执行审核、出纳复核、记账、冲销、作废等操作。页面以**分录行**为粒度展示凭证（每条分录占一行，同一凭证多行共用凭证字号），支持多标签视图、双行筛选、批量操作、打印、导出与推送至外账帐套。

本页属于财务凭证管理模块，需满足以下要求：
1. 关键操作均有二次确认，防止误操作
2. 已作废/已冲销凭证的状态变更操作不生效，系统跳过
3. 凭证字号为可点击链接，点击跳转至凭证录入页进行编辑

---

## 适用范围与前置配置

1. 适用对象：财务会计（审核）、出纳（复核）、财务主管（记账/冲销/作废）
2. 数据范围：当前账套下所有凭证（含手工录入、系统自动生成的期末结转凭证）
3. 前置配置：
   - 系统已完成会计科目配置（`AcctSubjects`）
   - 已有凭证数据写入 `sessionStorage["ManualVouchers"]`
   - 推送功能需在"帐套管理"中配置启用的子帐套（`AcctSetTree`）
4. 口径说明：
   - 凭证总金额 = `amount` 字段值（录入时取借方合计与贷方合计的较大值）
   - 审核人 = `auditUser` 字段；若为空但状态已是已审核/已记账/已复核/待记账，则显示"系统审核"，否则显示"-"
   - 外部系统数据单号仅首条分录行（`lineIndex === 0`）显示

## A. 工具栏按钮

| # | 按钮名称 | 样式 | 默认状态 | 触发条件 | 操作说明 |
|---|---------|------|---------|---------|---------|
| 1 | 反审核 | 默认 | 禁用 | 选中凭证 | confirm 确认后，将选中凭证状态改为"待审核"；已作废/已冲销跳过 |
| 2 | 审核 | 默认 | 禁用 | 选中凭证 | confirm 确认后，将选中凭证状态改为"已审核"；已作废/已冲销跳过 |
| 3 | 出纳复核 | 默认 | 禁用 | 选中凭证 | 弹出出纳复核确认弹窗；确认后写入 `cashierUser`，状态变"已复核" |
| 4 | 取消复核 | 默认 | 禁用 | 选中凭证 | confirm 确认后，清除 `cashierUser`，状态回退为"已审核" |
| 5 | 记账 | 默认 | 禁用 | 选中凭证 | confirm 确认后，将选中凭证状态改为"已记账"；已作废/已冲销跳过 |
| 6 | 取消记账 | 默认 | 禁用 | 选中凭证 | confirm 确认后，将已记账凭证状态回退为"已审核" |
| 7 | 删除 | 默认 | 禁用 | 选中凭证 | confirm "确认删除N张凭证？不可恢复" → 从 ManualVouchers 物理删除 |
| 8 | 操作▼ | 默认 | 始终可用 | 点击展开 | 下拉菜单：作废 / 取消作废 / 冲销 / 标错 / 取消标错 |
| 9 | 栏目 | 默认 | 始终可用 | 点击 | 打开栏目设置弹窗（27条字段配置） |
| 10 | 导出 | 默认 | 禁用 | 选中凭证 | 导出选中凭证为 JSON 文件（`凭证-{时间戳}.json`）|
| 11 | 推送 | **橙色高亮** | 禁用 | 选中凭证 | 推送选中凭证至外账帐套；须在帐套管理中配置子帐套 |
| 12 | 🖨（打印） | 默认 | 禁用 | 选中凭证 | 打开新窗口展示打印预览，含"打印"/"关闭"按钮 |

> **注意**：按钮1-7、10-12 带 `voucher-center__action` 样式，勾选凭证后激活，未选中时全部禁用。

## B. 筛选条件

### 第一筛选行

| 控件名称 | 控件类型 | 默认值 | placeholder / 选项 |
|---------|---------|--------|-------------------|

| 金额 | 数字范围（`id="voucher-amount-min"` / `id="voucher-amount-max"`）| 空 | 最小值 / 最大值 |

### 第二筛选行

| 控件名称 | 控件类型 | 默认值 | placeholder / 选项 |
|---------|---------|--------|-------------------|
| 科目编码 | 文本输入框（`id="voucher-subject-code"`）| 空 | 输入科目编码 |
| ○ 会计期间（单选，默认未选）| 期间范围下拉（`id="vaPeriodStart"` / `id="vaPeriodEnd"`）| 2026.01 ~ 2026.12 | 2022.06 / 2025.09 / 2026.01 |
| 单号（智能查询）| 文本输入框（`id="voucher-smart-search"`）| 空 | 输入单号智能查询：YD运单 / BX报销 / PC批次... |
| 查询 | 按钮 | — | 点击触发 `applyVoucherAuditSearch()`，重置页码为1 |

> **制单日期与会计期间互斥**：默认启用制单日期筛选，会计期间控件禁用；切换单选按钮后另一组启用。

### 智能单号查询识别规则

| 输入前缀 | 自动识别 | 过滤提示 |
|---------|---------|---------|
| `YD` | 运单类型 | `→ 运单` |
| `BX` | 报销单类型 | `→ 报销单` |
| `PC / APC / ASH` | 批次类型 | `→ 批次` |
| 其他 | 全类型模糊匹配 | `→ 全类型匹配` |

## G. 业务逻辑

### 1. 列表加载

1. 页面进入后，从 `sessionStorage["ManualVouchers"]` 加载全量凭证
2. 初始按**状态权重**排序：待审核(1) < 已审核(2) < 已记账/已过账(3) < 已冲销(5) < 已作废(6)
3. 将每张凭证展开为分录行：遍历 `v.lines[]`，`lineIndex = 0, 1, 2...`；若 `lines` 为空则生成单行
4. 默认激活"全部凭证"标签

### 2. 凭证状态机

```
待审核 ──[审核]──→ 已审核 ──[出纳复核]──→ 已复核 ──[记账]──→ 已记账
待审核 ←──[反审核]── 已审核
已审核 ←──[取消复核]── 已复核
已审核 ←──[取消记账]── 已记账
任意非终态 ──[作废]──→ 已作废
已记账 ──[冲销]──→ 已冲销 + 新增红字对冲凭证（status="已记账"）
```

**状态枚举**

| 状态值 | CSS 类 | 标签色 | 说明 |
|--------|--------|--------|------|
| 待审核 | `is-pending` | 黄色（`#fef9c3 / #a16207`）| 初始状态 |
| 已审核 | `is-audited` | 蓝色（`#dbeafe / #2563eb`）| 已通过审核 |
| 待复核 | `is-reviewing` | 橙色（`#fff7ed / #c2410c`）| 过渡状态（兼容旧数据）|
| 已复核 | `is-reviewed` | 青色（`#ecfeff / #0e7490`）| 出纳已签字 |
| 待记账 | `is-to-post` | 紫色（`#f5f3ff / #7c3aed`）| 过渡状态 |
| 已记账 | `is-posted` | 绿色（`#dcfce7 / #16a34a`）| 凭证已记账生效 |
| 已冲销 | `is-void` | 红色（`#fee2e2 / #dc2626`）| 已产生红字对冲凭证 |
| 已作废 | `is-void` | 红色（`#fee2e2 / #dc2626`）| 已作废，金额清零 |

### 3. 作废逻辑

1. confirm 确认后执行
2. 状态置为"已作废"
3. `amount` 置为 `"0.00"`
4. 所有分录 `debit = 0`，`credit = 0`
5. 备份原始数据至 `voidBackup = { amount, lines }` （用于审计追溯）
6. 记录 `voidedAt`（ISO 时间）、`voidedBy`（当前登录用户）
7. 已作废/已冲销凭证不再执行作废（跳过）

### 4. 冲销逻辑

1. confirm 确认后执行
2. 原凭证 status 置为"已冲销"
3. 生成新红字对冲凭证：
   - `id`：在原凭证 ID 基础上递增序号生成
   - `date`：当前日期（`new Date().toISOString().split('T')[0]`）
   - `status`：`"已记账"`（直接记账，无需再次审批）
   - `amount`：`-Math.abs(原金额)`
   - `isRed`：`true`（打印时红色字体）
   - `reverseOf`：原凭证 ID
   - `lines[]`：借贷金额全部取负；摘要更新为冲销说明
   - 摘要格式：`"冲销 M月N日第X号凭证：{原摘要}"`
4. 新红字凭证插入到 ManualVouchers 列表头部
5. 已冲销/已作废凭证不再执行冲销（跳过）

### 5. 批量操作规则

1. 勾选任意分录行后，同凭证所有分录行联动勾选（`toggleVoucherGroupSelection`）
2. 表头全选复选框控制当前页所有分录行
3. `getSelectedVoucherIds()` 按 `data-voucher-id` 去重，返回凭证 ID 数组（非分录行 ID）
4. 选中任意凭证后，所有 `voucher-center__action` 按钮从禁用变为可用

### 6. 推送至外账帐套

1. 从 `sessionStorage["AcctSetTree"]` 获取帐套树，找到当前帐套节点
2. 过滤 `status === 'enabled'` 的子帐套
3. 无子帐套 → alert：`"当前帐套没有已启用的子帐套，无法推送"`
4. 仅一个子帐套 → 自动选择目标
5. 多个子帐套 → 弹出选择弹窗
6. 执行凭证过滤（按模板规则）→ 推送至目标帐套的 ManualVouchers 快照

### 7. 打印

1. 校验选中凭证
2. 解析每张凭证的分录（科目编码+科目名称优先从 `AcctSubjects` 查找，兜底用内置科目字典 `BUILTIN`）
3. 在新窗口（`width=1100, height=800`）中渲染打印预览 HTML
4. 打印窗口含两个操作按钮：🖨 打印 / 关闭

## L. 验收标准

1. 列表正确按状态权重排序后展示，分录行展开正确（每张凭证 N 行）
2. 筛选条件中制单日期与会计期间正确互斥；智能单号识别前缀正确
3. 标签页过滤逻辑与状态机一致（含新旧状态兼容）
4. 同凭证多分录行勾选联动正确
5. 作废操作：金额/分录清零，voidBackup 保留原始数据，审计日志记录
6. 冲销操作：原凭证标记已冲销，新红字凭证金额正确取负，摘要格式正确
7. 出纳复核：确认后 cashierUser 写入当前登录用户，状态变"已复核"（不直接变"已记账"）
8. 外部系统数据单号渲染：最多3个链接，超出 +N，点击弹出居中弹窗
9. 凭证字号点击：正确跳转至凭证录入页并带入编辑 ID
10. 推送功能：无子帐套时有明确错误提示；有子帐套时正确推送
