## 页面概述

凭证录入页（模块编码：`VoucherEntryReview`）是财务管理系统的手工凭证制单入口，供会计人员录入借贷分录后提交审核。页面分为**头部表单区**（日期、凭证字、凭证号、附件数）和**分录明细区**（逐行录入摘要、科目、辅助核算、借贷金额），实时更新合计行，提交时校验借贷平衡。

页面兼顾**新增模式**（新建空白凭证）和**编辑模式**（从凭证审核页点击凭证字号跳转，回填已有数据）。

---

## 适用范围与前置配置

1. **适用对象**：会计（录入、修改草稿）；财务主管（修改待审核凭证）
2. **数据范围**：当前账套下手工凭证，写入 `sessionStorage["ManualVouchers"]`
3. **前置配置**：
   - 会计科目已配置（`AcctSubjects` in sessionStorage/localStorage）
   - 凭证字模式已配置（`VoucherWordMode` in localStorage）：`"ji"` 模式仅有"记"；默认模式有"收/付/转"
   - 摘要模板已配置（`getVoucherSummaryTemplates()`）
   - 辅助核算项目已配置（`AuxiliaryData:{key}` in sessionStorage/localStorage）
4. **口径说明**：
   - 凭证总金额 = 借方合计（`totalDebit`），保留2位小数
   - 凭证号格式 = 凭证字 + `-` + 4位序号（例：`转-0001`）；序号取现有凭证中该前缀最大序号+1
   - 保存状态统一为 `"待审核"`，无草稿状态持久化（草稿功能为占位）

## J. 业务逻辑

### 1. 初始化逻辑

1. 读取 AcctSubjects（sessionStorage → localStorage → ACCOUNTING_STANDARD_TEMPLATES 兜底）
2. 修正现金类科目的 `aux` 字段：若未含"现金流量项目"，自动追加
3. 初始化 10 条空白分录行（`veRows` 数组）
4. 自动生成凭证号（取 ManualVouchers 中最大数字序号+1）
5. 渲染分录表格（`veRenderEntryTable()`）

### 2. 实时合计更新

每当借方/贷方输入框变化（`oninput`）：
- 遍历 `veRows` 累加 `debit` 和 `credit`
- 更新 `veTotalDebit` / `veTotalCredit` 的文本内容（`toFixed(2)`）

### 3. 提交校验（`veDoSubmit`）

| 校验项 | 条件 | 失败提示 |
|--------|------|---------|
| 有效分录行数 | `validRows.length < 2`（科目非空的行数）| "至少需要2行有效分录" |
| 借贷平衡 | `Math.abs(debit - credit) > 0.001` | "借贷不平衡，请检查分录金额" |
| 通过后 | 调用 `veSaveVoucher()` 保存，状态"待审核" | Toast"凭证已提交，进入审核流程" |

### 4. 保存逻辑（`veSaveVoucher`）

1. 读取表单字段（日期/凭证字/凭证号/附件数）
2. 凭证号：用户输入为准；为空则自动生成；补齐4位
3. 组装凭证ID = `凭证字-凭证号`（如 `转-3527`）
4. 过滤有效分录行（摘要/科目/借贷至少一项非空）
5. 计算 `totalDebit`（累加所有行 debit）
6. 构造凭证对象并 push 到 ManualVouchers（不替换同ID已有凭证，每次都新增）
7. 更新凭证号输入框显示

> **注意**：`veSaveVoucher` 不会校验借贷平衡，也不检查重复ID（每次保存都 push 新记录）。多次保存同一凭证会产生多条记录。"保存"按钮上的逻辑依赖上层调用，`veDoSubmit` 有校验，`veDoSave` 无校验。

### 5. 编辑模式（从凭证审核页跳转）

1. 调用 `loadVoucherForEdit(id)`（在 voucher.js 中定义）
2. 若凭证状态为"已记账"/"已过账"，弹出提示不可修改
3. 回填日期/凭证字/凭证号到表单
4. 清空分录表，逐行回填 lines[]（调用 `addVoucherLineRow`）
5. 编辑模式下，凭证号输入框不会因凭证字切换而重新生成

### 6. 凭证字切换行为

- 切换凭证字时：更新中央标题（`veVoucherTitle`）
- 若凭证号为空，自动填入该凭证字的下一个序号

### 7. 新建表单清空逻辑（`veResetForm`）

- 重置 `veRows` 为2条空白行
- 清空凭证号输入框
- 重新渲染表格
- 调用 `veUpdateVoucherNo()` 自动生成新凭证号

## O. 验收标准

1. 页面加载后自动填入当日日期、自动生成凭证号，默认10行空白分录
2. 凭证字切换后，中央标题同步更新（收款/付款/转账凭证）
3. 摘要和科目输入框的自动补全：触发、过滤、选中、关闭均正常
4. 辅助核算浮层：选项按科目aux配置展示；多选勾选后按钮文本正确更新；无配置时显示提示
5. 行右键菜单（序号格点击）：增行/插行/复制行/删行功能正常；删行后最少保留1行
6. 借方/贷方实时更新合计行
7. 提交校验：有效分录<2行或借贷不平衡时 Toast 错误，不进行保存
8. 保存：凭证写入 ManualVouchers，status="待审核"，id格式正确
9. 常用凭证参照：列表显示正确，过滤/引入功能正常，有内容时引入前确认
10. 通用确认弹窗：清空/新增均先弹确认，取消不执行操作
