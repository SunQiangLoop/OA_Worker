## 页面概述

期初余额录入页（模块编码：`FinanceOpeningBalance`）是财务管理系统的账套初始化核心入口，供财务人员在建账初期录入所有会计科目的期初余额。页面以会计科目树形列表展示所有科目，末级科目支持直接录入期初余额，父科目余额由子科目自动汇总计算，底部实时显示借贷双方合计及试算平衡状态。

本页属于财务基础配置模块，需满足以下要求：
1. 期初余额数据准确性要求极高，录入后须通过试算平衡校验方可进行后续账务处理
2. 父科目余额不可手工录入，必须由系统自动汇总末级科目，防止数据不一致
3. 支持导入/导出便于批量操作，清零需二次确认防止误操作

---

## 适用范围与前置配置

1. 适用对象：财务主管、会计（建账阶段）
2. 数据范围：当前账套下所有已维护的会计科目
3. 前置配置：
   - 系统已在【会计科目】模块完成科目表维护（`AcctSubjects`）
   - 已配置科目级次规则（`SubjectCodeSetting.lengths`，默认 `[4, 2, 2]` 即4-2-2位编码）
4. 口径说明：
   - 试算平衡口径：仅末级科目（叶子科目）参与汇总；差额绝对值 < 0.005 视为平衡
   - 父科目判断规则：当前科目编码是其他任意科目编码的前缀，则视为父科目
   - 年初余额：当前版本固定显示 `0.00`，功能待完善

## 业务逻辑

### 1. 科目表加载

1. 页面进入后，从 `sessionStorage["AcctSubjects"]` 读取科目表；若为空则尝试 `localStorage["AcctSubjects"]`
2. 科目表按 `code` 字段升序排列，保证父子层级显示顺序正确
3. 若科目表仍为空，表格显示"暂无科目数据，请先在【会计科目】中维护科目表"
4. 每条科目计算 `isLeaf`：当 `allCodes` 中无其他编码以该编码为前缀时，`isLeaf = true`

### 2. 科目级次判断

1. 读取 `SubjectCodeSetting.lengths`（从 sessionStorage 或 localStorage），默认 `[4, 2, 2]`
2. 累计每级编码长度：4→一级，6→二级，8→三级，以此类推
3. 级次徽章颜色：一级蓝（`#e6f0ff / #2c5fad`），二级绿（`#e6fff0 / #1a7a4a`），三级橙（`#fff7e6 / #a05a00`），四级及以上紫（`#f9f0ff / #6a1fa0`）

### 3. 期初余额录入规则

1. **末级科目**：期初余额输入框可编辑（`type="number"`），输入时实时触发：
   - 更新当前科目 `balance`
   - 向上汇总所有祖先父科目 `balance`（递归刷新父科目禁用输入框显示值）
   - 自动保存到 sessionStorage
   - 重新计算试算平衡
2. **父科目**：期初余额输入框禁用（`readonly disabled`），值 = 所有直接+间接子科目 `balance` 之和；输入框下方显示"汇总子科目"灰色小字提示
3. 科目名称缩进规则：级别 N > 1 时，左侧缩进 `(N-1) × 20 + 16` px

### 4. 试算平衡计算

```
借方合计 = Σ{ sub.balance | sub.isLeaf && sub.direction === "借" }
贷方合计 = Σ{ sub.balance | sub.isLeaf && sub.direction === "贷" }
差额 = 借方合计 - 贷方合计

if |差额| < 0.005:
    状态 → "✅ 试算平衡"（绿色背景 #f6ffed）
else:
    状态 → "⚠ 试算不平衡 | 差额: {|差额|}"（红色背景 #ffebee）
```

> 注意：父科目不参与试算，避免重复计入。

### 5. 保存与持久化

1. 每次末级科目输入时**自动保存**（`persistOpeningBalances()`）
2. 点击"保存"按钮时再次显式持久化并弹出成功提示
3. 存储格式：`sessionStorage["OpeningBalances"] = [{code, name, direction, balance}, ...]`（全量覆盖）
4. 页面再次进入时，自动读取已保存余额并合并到科目列表（`loadOpeningBalances()`）

### 6. 导入导出规范

#### 6.1 导出
1. 导出格式：TSV（`.tsv`，Tab 分隔，UTF-8 BOM）
2. 导出文件名：`期初余额.tsv`
3. 导出字段顺序：科目编码 / 科目名称 / 方向 / 期初余额
4. 含表头行；科目全量导出（含父科目）

#### 6.2 导入（计划功能，当前版本为占位）
1. 计划支持 Excel（`.xlsx`）或 CSV/TSV 格式
2. 导入前校验：文件格式、表头名称与顺序、数值格式
3. 导入策略：按科目编码匹配，覆盖现有余额；未匹配科目跳过
4. 导入模板字段（建议）：

| 模板列序 | 模板表头 | 是否必填 | 类型 | 示例值 | 填写规则 |
|---------|---------|---------|------|--------|---------|
| 1 | 科目编码 | 是 | String | `1001` | 必须与系统科目编码完全一致 |
| 2 | 期初余额 | 是 | Decimal | `58000.00` | 数字，保留2位小数；父科目填入值将被系统覆盖 |

## 验收标准

1. 科目表正确从 AcctSubjects 加载，按编码升序排列，层级缩进正确
2. 末级科目输入余额后，所有祖先父科目的汇总值实时刷新，无延迟
3. 试算平衡计算结果仅统计末级科目，不重复计入父科目
4. 保存后刷新页面，期初余额正确回填
5. 清零需二次确认，清零后所有余额归零，底部试算状态同步更新
6. 导出文件内容与页面数据一致，字段顺序符合规范
7. 父科目期初余额输入框不可编辑，显示"汇总子科目"提示
